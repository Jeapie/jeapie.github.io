Server API Overview

The Jeapie Server API serves two purposes:

  1. Programmatically delivering notifications from your server or from one mobile device to another.
  2. Adding, updating, of fetching data for users of your app without using our device SDKs.

Our Server API allows you to programmatically view, update, and create the following:

  • Apps
  • Players (Also known as Devices, or Users)
  • Notifications

API Libraries

Our server API can be access directly using the regular HTTP library of your choice and does not require any special libraries.

Authentication

Some API methods require User or App authentication REST API Keys. They are:

    1. Jeapie App creation or modification - Requires your Jeapie 'User REST API Key'.
    1. Notification Creation - Requires your Jeapie 'App REST API Key' when specifying targets using "tags" or "included_segments". Otherwise no token is required.

Security Warning!

Jeapie User and App REST API Keys should be treated like a password and not stored in your app!

  • The 'User REST API Key' is visible on the "Account Management" page under the "API Keys" tab.
  • The 'App REST API Key' is under the "Application Settings" page under the "API Keys" tab.

Apps

A Jeapie app represents a single app across all of the platforms it is released on, including Android, iOS, and Windows Mobile.

Jeapie User API Key required

In order to call any of the "Apps" related API methods, you must supply your Jeapie user token.

Available /apps methods

    1. Fetch the details of all of your apps GET /apps
    1. Fetch the details of one of your apps GET /apps/:id
    1. Create an app POST /apps
    1. Update the details of an app PUT /apps:id

Players (Also known as Devices, or Users)

A Jeapie app can have many users, each of which corresponds to a unique user that has installed and run your application.

A user has the following attributes:

  • app_id
  • id
  • device_type
  • identifier
  • language
  • session_count
  • last_active
  • device_os
  • timezone
  • ad_id
  • badge_count
  • app_version
  • created_at
  • amount_spent
  • tags
  • duration
  • invalid_identifier
  • badge_count

The *id* and *created_at* values are assigned by the Jeapie API when a user is created.

*app_id* is available on the Jeapie website after creating a new app.

*invalid_identifier* is set to true in the following cases:

  1. The user does not have an identifier, for instance if they declined the push notification permission.
  2. If the user re-installed your app and a new user record was created for them
  3. If they failed to receive a push notification because they uninstalled the app.

Available /players methods

  • Fetch the details of a user GET /players/:id
  • Add a new user to a Jeapie app POST /players
  • Update the details of a user PUT /players/:id
  • Indicate a user started using your app POST /players/:id/on_session
  • Indicate a user made a purchase in your app POST /players/:id/on_purchase
  • Update the total session time of a user POST /players/:id/on_focus

Notifications

The notifications API can be used to send notifications to individual users, segments, or users that match specific tags. It can also be used to deliver a single notification to multiple Jeapie apps.

Available /notifications methods

  • Fetch the details of notifications that have been sent GET /notifications
  • Fetch the details of a notification GET /notifications/:id
  • Send a notification to a user, segment, app, or multiple apps POST /notifications
  • Track that a notification was opened
