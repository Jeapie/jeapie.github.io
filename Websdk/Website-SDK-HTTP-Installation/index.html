Website SDK HTTP Installation

Jeapie SDK Installation for Chrome websites (desktop + mobile)

HTTP vs. HTTPS

This is the guide for Chrome push on websites that have some pages served via HTTP instead of HTTPS.

If possible, we recommend making sure all your pages are only served via HTTPS and then following this guide instead: Website SDK HTTPS Installation

Requirements

W3C Web Push Notifications are currently only support by Chrome 42+

  • Includes Chrome for Windows, Mac OSX, Linux, Chrome OS, and Android but not iOS.

1. Include Required JeapieSDK.js

1.1 Include https://cdn.jeapie.com/sdks/JeapieSDK.js in the the <head> HTML tag of each of your website pages. The best way is to add these to the code that generates the layout for each of your webpages. The resulting HTML should be the following:

<head>
  <script src="https://cdn.jeapie.com/sdks/JeapieSDK.js" async></script>
</head>

2. Jeapie HTTP Settings

2.1 Go to Jeapie.com and select "Application Settings". Next press the Configure button on "Chrome Website (GCM)" and fill in the 3 required HTTP settings fields.

  • Subdomain - Enter any name you would like your users to see as a subdomain requesting push notification permissions. Example if you entered jeapieexample they will see "s-jeapieexample.jeapie.com would like to use your push notifications.".

  • Your Site URL - Enter your website's URL here so our SDK can validate your site is requesting push notification permissions. You may enter localhost or * here in your testing if needed. WARNING: Make sure you DO NOT have * in the field when you deploy Jeapie on your live website!

  • Google Project number - Enter your Google Project number found in the Google Developer Console.

3. Initialize Jeapie

3.1 Call Jeapie.init from a javascript file that is included on every page.

  • Update 5eb5a37e-b458-11e3-ac11-000c2940e62c with your Jeapie AppId.
  • Update "s-jeapieexample" with the Subdomain value you entered in our dashboard prefixed with 's-'.
var Jeapie = Jeapie || [];

Jeapie.push(["init", {appId: "5eb5a37e-b458-11e3-ac11-000c2940e62c", subdomainName: "s-jeapieexample"}]);

4. Call registerForPushNotifications

4.1 Add a button or link to your page to turn on push notifications. Call registerForPushNotifications on Jeapie from the button press to prompt for permission.

function registerForPush() {
  Jeapie.push(["registerForPushNotifications"]);
}

Jeapie.push(function() {
  if (Jeapie.isPushNotificationsSupported())
    document.getElementById("registerForPushButton").onclick = registerForPush;
  else // Show a notifications not supported message if you like.
    document.getElementById("not-supported-message").style.display = 'block';
});

That's It!

That's it for the setup. See our Web SDK API for more functions and our examples on our Github page.
