Website SDK HTTPS Installation

Jeapie SDK Installation for Chrome websites (desktop + mobile)

Requirements

W3C Web Push Notifications are currently only support by Chrome 42+

  • Includes Chrome for Windows, Mac OSX, Linux, Chrome OS, and Android but not iOS.

HTTP and HTTPS

If some of your pages are served via HTTP instead of HTTPS then you will need to follow our HTTP Installation Guide instead.

If possible, we encourage you to migrate all your pages to HTTPS first and then continue using this guide.

1. Download the SDK

1.1 Download the latest Jeapie Chrome Web SDK.

1.2 Copy the 3 files from SetupFiles out of the directory and into the top level root of your site directory.

2. Include Required Files

2.1 Include https://cdn.jeapie.com/sdks/JeapieSDK.js and manifest.json in the <head> HTML tag of each of your website pages. The best way is to add these to the code that generates the layout for each of your webpages. The resulting HTML should be the following:

<head>
  <script src="https://cdn.jeapie.com/sdks/JeapieSDK.js" async></script>
  <link rel="manifest" href="manifest.json">
</head>

3. Update manifest.json

3.1 Open manifest.json and change the name and short_name to your site's name.

3.2 Update gcm_sender_id with your Google Project Number. You can find this number from the Google Developers Console.

"gcm_sender_id": "703322744261",

4. Initialize Jeapie

4.1 Call Jeapie.init from a javascript file that is included on every page. Update 5eb5a37e-b458-11e3-ac11-000c2940e62c with your Jeapie AppId.

var Jeapie = Jeapie || [];

Jeapie.push(["init", {appId: "5eb5a37e-b458-11e3-ac11-000c2940e62c"}]);

Important

You must keep all 3 files SDK together. They must be placed at the root of your site directory.

https://cdn.jeapie.com/sdks/JeapieSDK.js and manifest.json must be located in the head html tag in every one of your pages. This is so any page can subscribe for notifications, any page can be set to be opened from a notification, changes to the Google Registration id can be updated, and session count can be accurately counted.

That's It!

That's it for the setup. See our Web SDK API for more functions and our examples on our Github page.
